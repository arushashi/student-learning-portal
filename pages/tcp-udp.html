Definition

What are TCP and UDP?

TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are two core transport layer protocols that define how data is transmitted over networks. TCP provides reliable, ordered delivery while UDP offers fast, connectionless communication.

Key Insight: TCP is like registered mail with tracking and confirmation, while UDP is like regular mail - faster but no delivery guarantees.

Simple Explanation

Understanding the Concept

Imagine you need to send important documents to someone. You have two options:

TCP - The Courier Service:

  • You establish a connection first (call the courier)
  • Package is tracked every step of the way
  • Recipient signs for delivery
  • If package is lost, it's resent
  • Packages arrive in the order sent
  • Reliable but slower and more expensive

UDP - The Postal Service:

  • You just drop the letter in a mailbox
  • No tracking or delivery confirmation
  • Letters might arrive out of order
  • Some letters might get lost
  • Fast and cheap but no guarantees

TCP (Transmission Control Protocol)

Key Features:

  • Connection-oriented: Establishes connection before data transfer
  • Reliable: Guarantees delivery through acknowledgments and retransmission
  • Ordered: Data arrives in the same order it was sent
  • Error Checking: Checks for corrupted data
  • Flow Control: Prevents sender from overwhelming receiver
  • Congestion Control: Prevents network overload

Three-Way Handshake:

  • SYN: Client requests connection
  • SYN-ACK: Server acknowledges and requests connection
  • ACK: Client acknowledges, connection established

UDP (User Datagram Protocol)

Key Features:

  • Connectionless: No connection establishment
  • Unreliable: No delivery guarantees
  • No Order: Packets may arrive out of order
  • Fast: Minimal overhead
  • No Flow Control: Can overwhelm receiver
  • No Congestion Control: Can cause network issues

Real-World Example

Video Streaming

Video streaming services like Netflix use UDP for streaming video. Losing a few frames is acceptable, but maintaining speed is crucial. For account login and payment, they switch to TCP to ensure reliability.

Key Points & Summary

TCP Use Cases

Web browsing, email, file transfer, database connections - where reliability is critical.

UDP Use Cases

Video streaming, gaming, DNS, VoIP - where speed is more important than reliability.

Port Numbers

Both protocols use ports to identify applications; TCP ports are separate from UDP ports.

Performance

UDP is faster due to lower overhead; TCP is slower but more reliable.

Quick Summary

  • TCP: Reliable, ordered, connection-oriented communication
  • UDP: Fast, connectionless, unreliable communication
  • TCP uses three-way handshake to establish connections
  • UDP has minimal overhead for better performance
  • Choose based on reliability vs speed requirements

Interview Questions & Answers

What's the difference between TCP and UDP?

TCP (Transmission Control Protocol):

  • Connection-oriented with three-way handshake
  • Reliable delivery with acknowledgments and retransmission
  • Ordered data delivery
  • Error checking and flow control
  • Slower due to overhead

UDP (User Datagram Protocol):

  • Connectionless - no handshake
  • Unreliable - no delivery guarantees
  • No ordering guarantees
  • No error checking or flow control
  • Faster due to minimal overhead

When would you choose TCP over UDP?

Choose TCP when:

  • Reliability is critical: File transfers, email, database connections
  • Data integrity matters: Financial transactions, authentication
  • Order is important: Web pages, document streaming
  • Error recovery needed: Important business communications

Examples: HTTP/HTTPS, SMTP, FTP, SSH, database connections

When would you choose UDP over TCP?

Choose UDP when:

  • Speed is priority: Real-time applications
  • Some data loss is acceptable: Video/audio streaming
  • Low latency required: Online gaming, live broadcasts
  • Broadcast/multicast needed: Sending to multiple recipients
  • Simple request-response: DNS queries

Examples: Video streaming, VoIP, online gaming, DNS, live broadcasts

How does TCP ensure reliable data delivery?

TCP ensures reliability through:

  • Sequence Numbers: Track packet order
  • Acknowledgments: Receiver confirms received packets
  • Retransmission: Resend lost or corrupted packets
  • Checksums: Detect data corruption
  • Flow Control: Prevent sender overload
  • Congestion Control: Prevent network overload
