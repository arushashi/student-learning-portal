Fault tolerance and continuous system operation

Definition

What is High Availability?

High availability is system design approach that ensures a system remains operational with minimal downtime, typically targeting 99.9%+ uptime through redundancy and failover mechanisms.

Key Insight: High availability eliminates single points of failure through redundancy and automatic failover.

Simple Explanation

Understanding the Concept

Think of a hospital emergency room — it must be available 24/7, no exceptions. If one doctor becomes unavailable, another immediately steps in. High availability in system design follows the same principle: the system is designed so that if any component fails, backup components seamlessly take over to keep everything running.

How High Availability Works

1. The "Nines" of Availability

  • 99% (Two Nines): ~3.65 days of downtime per year
  • 99.9% (Three Nines): ~8.76 hours of downtime per year
  • 99.99% (Four Nines): ~52.6 minutes of downtime per year
  • 99.999% (Five Nines): ~5.26 minutes of downtime per year

2. Key HA Patterns

  • Active-Passive (Failover): A standby node takes over when the active node fails
  • Active-Active: Multiple nodes handle traffic simultaneously; if one fails, others absorb the load
  • N+1 Redundancy: Deploy one extra component beyond what is needed to handle the load
  • Geographic Redundancy: Distribute systems across multiple data centers or regions

HA Components

Load Balancers: Distribute traffic and detect unhealthy nodes

Health Checks: Continuously monitor system components for failures

Auto-scaling: Dynamically add or remove instances based on demand

Data Replication: Keep data synchronized across multiple nodes

Real-World Example

AWS Multi-AZ Deployments

AWS provides high availability through Multi-AZ (Availability Zone) deployments. Services like RDS automatically replicate data to a standby instance in a different AZ, enabling automatic failover within minutes if the primary instance fails.

Key Points & Summary

Availability Nines

Each additional nine dramatically reduces allowed downtime, from days to minutes per year.

Failover Strategies

Active-passive and active-active patterns provide different trade-offs between cost and recovery time.

Redundancy

Eliminating single points of failure through redundant components at every layer of the stack.

Health Monitoring

Continuous health checks and automated alerting enable rapid detection and response to failures.

Quick Summary

  • High availability targets minimal downtime through redundancy and failover
  • Five nines (99.999%) allows only ~5 minutes of downtime per year
  • Active-passive uses standby nodes; active-active distributes load across all nodes
  • Redundancy at every layer eliminates single points of failure
  • Health checks and monitoring are essential for detecting and responding to failures

Interview Questions & Answers

What are the key principles of high availability?

Key principles include redundancy (multiple components), failover (automatic switching), load balancing, monitoring, disaster recovery, and regular testing. These ensure continuous operation during failures.

What's the difference between high availability and fault tolerance?

High availability focuses on minimizing downtime through redundancy, while fault tolerance enables the system to continue operating despite failures. Both are essential for reliable systems.

How do you design a system for 99.999% availability?

Achieving five-nines availability requires a comprehensive approach:

  • Multi-region deployment: Distribute across geographically separate data centers
  • Automated failover: Zero-downtime switching with health check-driven automation
  • Redundancy at every layer: Duplicate load balancers, app servers, databases, and network paths
  • Chaos engineering: Regularly test failure scenarios to validate resilience

What is the difference between active-passive and active-active failover?

These are two fundamental high availability patterns:

  • Active-Passive: One node handles all traffic while a standby node waits idle; failover switches traffic to the standby when the active node fails
  • Active-Active: All nodes handle traffic simultaneously; if one fails, remaining nodes absorb the additional load
  • Trade-offs: Active-active offers better resource utilization and lower latency but is more complex to implement
  • Data considerations: Active-active requires data synchronization strategies to maintain consistency
