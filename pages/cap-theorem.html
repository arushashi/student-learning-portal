CAP Theorem

Fundamental trade-offs in distributed database systems

Advanced â± 25 min

Definition

CAP theorem states that a distributed system can only provide two of three guarantees: Consistency, Availability, and Partition Tolerance.

The Three Properties

  • Consistency: All nodes see the same data simultaneously
  • Availability: System remains operational despite failures
  • Partition Tolerance: System continues operating despite network partitions

Trade-offs

  • CP: Consistency + Partition Tolerance (sacrifice availability)
  • AP: Availability + Partition Tolerance (sacrifice consistency)
  • CA: Consistency + Availability (no partition tolerance)
