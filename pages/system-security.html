System Security

Security fundamentals and protection mechanisms

Definition

What is System Security?

System security encompasses the protection of systems, networks, and data from unauthorized access, use, disclosure, disruption, modification, or destruction.

Key Insight: Security requires defense in depth with multiple layers of protection.

Simple Explanation

Understanding the Concept

Think of a medieval castle. It doesn't rely on just one wall — it has a moat, outer walls, inner walls, locked gates, and guards at every level. System security works the same way: multiple layers of defense ensure that if one barrier is breached, others still protect what's inside.

Core Security Principles (CIA Triad)

1. Confidentiality

  • Ensuring data is accessible only to authorized parties
  • Encryption protects data in transit (HTTPS/TLS) and at rest (AES)
  • Access controls restrict who can view sensitive information

2. Integrity

  • Ensuring data is accurate and hasn't been tampered with
  • Hashing and digital signatures verify data authenticity
  • Input validation prevents injection of malicious data

3. Availability

  • Ensuring systems remain accessible when needed
  • DDoS protection and rate limiting prevent service disruption
  • Redundancy and failover maintain uptime during attacks

Common Security Threats

SQL Injection: Attackers insert malicious SQL into input fields to manipulate database queries, potentially reading, modifying, or deleting data.

Cross-Site Scripting (XSS): Malicious scripts are injected into web pages viewed by other users, enabling session hijacking and data theft.

Cross-Site Request Forgery (CSRF): Tricks authenticated users into submitting unwanted requests to a web application they're logged into.

Man-in-the-Middle (MitM): Attackers intercept communication between two parties to eavesdrop or alter data in transit.

Defense Mechanisms

  • HTTPS/TLS: Encrypts all data in transit between client and server
  • CORS (Cross-Origin Resource Sharing): Controls which domains can access your API
  • Content Security Policy (CSP): Prevents XSS by restricting resource loading
  • Rate Limiting: Throttles requests to prevent brute-force and DDoS attacks
  • Input Validation & Parameterized Queries: Prevents injection attacks
  • Defense in Depth: Multiple overlapping security layers so no single failure compromises the system

Real-World Example

Google's Zero Trust Security Model

Google implements a "BeyondCorp" zero trust architecture where no network is considered trusted. Every request is authenticated and authorized regardless of its origin. This approach uses device certificates, user identity verification, and context-aware access policies to secure every interaction — eliminating the traditional concept of a trusted internal network.

Key Points & Summary

Defense in Depth

Multiple security layers (network, application, data) ensure that breaching one layer doesn't compromise the entire system.

Encryption

HTTPS encrypts data in transit while AES and similar algorithms protect data at rest, maintaining confidentiality throughout the data lifecycle.

Input Validation

Sanitizing and validating all user input prevents injection attacks (SQL, XSS) that exploit trust in user-supplied data.

Zero Trust

Never trust, always verify — authenticate and authorize every request regardless of its network origin.

Quick Summary

  • The CIA triad (Confidentiality, Integrity, Availability) guides security design
  • HTTPS/TLS encrypts data in transit to prevent eavesdropping
  • SQL injection and XSS are among the most common web vulnerabilities
  • CORS policies control cross-origin resource access
  • Defense in depth uses multiple overlapping security layers

Interview Questions & Answers

What are the core principles of security?

Core security principles include confidentiality (data privacy), integrity (data accuracy), availability (system uptime), authentication (user verification), authorization (access control), and non-repudiation (transaction tracking).

What is defense in depth?

Defense in depth is a layered security approach where multiple security controls are applied throughout the system. If one layer fails, others continue to provide protection, creating comprehensive security coverage.

How do you prevent SQL injection attacks?

SQL injection can be prevented through multiple techniques:

  • Parameterized queries: Use prepared statements that separate SQL logic from data
  • ORM frameworks: Use object-relational mappers that auto-parameterize queries
  • Input validation: Whitelist acceptable input patterns and reject everything else
  • Least privilege: Database accounts should have minimal required permissions

What is the difference between authentication and authorization?

Authentication and authorization serve different purposes:

  • Authentication (AuthN): Verifies who the user is (identity) — e.g., login with username/password, OAuth, biometrics
  • Authorization (AuthZ): Determines what the user can do (permissions) — e.g., role-based access control (RBAC), policies
  • Order: Authentication always happens first; authorization follows
  • Example: Logging into your email (authentication) vs. having admin rights to manage other accounts (authorization)
