Definition

What is HTTP?

HTTP (Hypertext Transfer Protocol) is the foundation of data communication on the World Wide Web. It's a request-response protocol that enables clients (browsers) to request resources from servers and receive responses. HTTP is the protocol that powers virtually all web browsing and API interactions.

Key Insight: HTTP is like the language of the internet - it defines how web browsers and servers talk to each other to deliver web pages, images, videos, and API data.

Simple Explanation

Understanding the Concept

Think of HTTP like ordering food at a restaurant. You (the client) look at the menu (web page), decide what you want, and tell the waiter (HTTP request) your order. The waiter goes to the kitchen (server), brings back your food (HTTP response), and you enjoy your meal!

HTTP Request-Response Cycle

1. Client Makes Request

  • Browser creates HTTP request with method, URL, headers, and optional body
  • Request sent over TCP connection to server
  • Server receives and processes the request

2. Server Sends Response

  • Server processes request and generates response
  • Response includes status code, headers, and body
  • Response sent back to client over TCP
  • Client processes response and displays content

HTTP Methods

GET: Retrieve data (safe, idempotent)

POST: Create new resource

PUT: Replace entire resource (idempotent)

PATCH: Partial update of resource

DELETE: Remove resource (idempotent)

HEAD: Get headers only (like GET but no body)

OPTIONS: Get allowed methods for resource

HTTP Status Codes

2xx Success:

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 204 No Content - Success but no content to return

3xx Redirection:

  • 301 Moved Permanently - Resource permanently moved
  • 302 Found - Resource temporarily moved
  • 304 Not Modified - Use cached version

4xx Client Error:

  • 400 Bad Request - Invalid request
  • 401 Unauthorized - Authentication required
  • 403 Forbidden - Access denied
  • 404 Not Found - Resource doesn't exist
  • 429 Too Many Requests - Rate limited

5xx Server Error:

  • 500 Internal Server Error - Generic server error
  • 502 Bad Gateway - Invalid response from upstream
  • 503 Service Unavailable - Server temporarily down

HTTP Headers

Request Headers:

  • Host - Domain name of server
  • User-Agent - Client information
  • Accept - Content types client can handle
  • Authorization - Authentication credentials
  • Content-Type - Format of request body

Response Headers:

  • Content-Type - Format of response body
  • Content-Length - Size of response body
  • Cache-Control - Caching instructions
  • Set-Cookie - Set cookies for client
  • Location - URL for redirection

Real-World Example

Web Browser Request

When you visit google.com, your browser sends: GET / HTTP/1.1 with headers like User-Agent and Accept. Google responds with status 200, headers including Content-Type: text/html, and the HTML content that renders the page.

Key Points & Summary

Stateless Protocol

Each request is independent; servers don't maintain client state between requests.

HTTP Versions

HTTP/1.0, HTTP/1.1, HTTP/2, and HTTP/3 with improvements in performance and features.

HTTPS

HTTP over TLS/SSL for encrypted communication, ensuring data privacy and integrity.

Cookies

HTTP cookies enable state management by storing data on client for session tracking.

Quick Summary

  • HTTP is request-response protocol for web communication
  • Uses methods like GET, POST, PUT, DELETE for operations
  • Status codes indicate request success or failure
  • Headers provide metadata about requests and responses
  • Stateless but can maintain state via cookies and sessions

Interview Questions & Answers

What is HTTP and how does it work?

HTTP (Hypertext Transfer Protocol) is the foundation of web communication. It works through:

  • Request-Response Model: Client sends request, server responds
  • Stateless Protocol: Each request is independent
  • TCP-based: Reliable transport layer
  • Text-based: Human-readable protocol
  • Application Layer: Layer 7 in OSI model

Typical flow: Client creates HTTP request â†’ Server processes â†’ Server sends response â†’ Client processes response.

What's the difference between HTTP and HTTPS?

HTTP (Hypertext Transfer Protocol):

  • Unencrypted communication
  • Data sent in plain text
  • Vulnerable to eavesdropping
  • Uses port 80 by default
  • Faster due to no encryption overhead

HTTPS (HTTP Secure):

  • Encrypted with TLS/SSL
  • Data protected from interception
  • Authentication via certificates
  • Uses port 443 by default
  • Slower due to encryption overhead

What are idempotent HTTP methods?

Idempotent methods produce the same result regardless of how many times they're called:

Idempotent Methods:

  • GET: Always returns same resource
  • PUT: Replacing resource with same content
  • DELETE: Deleting same resource twice has same effect
  • HEAD: Same as GET but no body
  • OPTIONS: Same options returned

Non-Idempotent Methods:

  • POST: Creates new resource each time
  • PATCH: May have different effects based on current state

What's the difference between PUT and PATCH?

PUT:

  • Replaces entire resource
  • Idempotent operation
  • Requires complete resource representation
  • Client must send all fields
  • Use when updating entire resource

PATCH:

  • Partially updates resource
  • Not necessarily idempotent
  • Requires only changed fields
  • Client sends only what changed
  • Use for partial modifications
